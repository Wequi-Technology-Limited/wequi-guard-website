4. App Permissions & Why We Need Them

Wequi Guard requires certain Android permissions to provide its core functionality. Here's why each permission is necessary:

Accessibility Service

Why we need it: The Accessibility Service is used to help enforce restrictions and detect when restricted screens or actions are accessed. This enables features such as:

  • Detecting when a blocked app or website is being accessed
  • Preventing bypass attempts of content restrictions
  • Showing warning or confirmation dialogs when users attempt restricted actions
Important Disclosure:

The Accessibility Service is NOT intended to read or collect personal messages, emails, or other private content. It is used solely to monitor app and website access for enforcement purposes.

Draw Over Other Apps (Overlay Permission)

Why we need it: This permission allows the Application to display warning screens, confirmation dialogs, and blocking overlays when restricted content or apps are accessed. This is essential for the blocking and enforcement features to work effectively.

Usage Access Statistics

Why we need it: This permission enables the Application to monitor which apps are being used, allowing us to enforce app-blocking rules and time limits (if this feature is enabled).

Device Admin

Why we need it: Device Admin capabilities may be requested to enhance security and prevent unauthorized uninstallation of protection features (if this feature is implemented).

User Consent and Disclosure:

Before enabling any of these permissions, the Application provides clear, in-app disclosures explaining what each permission does and why it is needed. You can review and modify these permissions at any time in your device's system settings.
